## 🛑 El Error del Junior

"Es alguien que sabe de todo"—vago y sin estructura. No distingue qué capas cubre un fullstack real ni qué nivel de profundidad se espera en cada una.

## 🧠 El Enfoque Curse Dev (Contra-pregunta)

* **Tú:** *"The definition of fullstack varies by company, so I'd love to understand what it means in your context. But from my perspective, would it help if I described the layers I'm comfortable working across?"*

## 💻 La Respuesta Técnica

Un **desarrollador fullstack** puede trabajar en todas las capas de una aplicación web: desde la interfaz de usuario hasta la lógica del servidor, la base de datos y la integración entre ellas.

**Las capas del stack:**

```
┌─────────────────────────────────┐
│  FRONTEND (Cliente)             │  → React, Next.js, HTML/CSS
│  Lo que el usuario ve e interactúa│
├─────────────────────────────────┤
│  BACKEND (Servidor)             │  → FastAPI, Node.js, Django
│  Lógica del negocio y APIs      │
├─────────────────────────────────┤
│  BASE DE DATOS                  │  → PostgreSQL, MongoDB, Redis
│  Persistencia de información    │
├─────────────────────────────────┤
│  INFRAESTRUCTURA / DEVOPS       │  → Docker, CI/CD, Variables de entorno
│  Cómo se despliega y opera      │
└─────────────────────────────────┘
```

**Lo que diferencia a un fullstack real:**
- No es experto en todo, pero puede trabajar en todas las capas
- Entiende cómo cada capa afecta a las demás
- Puede hacer un feature de punta a punta sin depender de otro equipo
- Comunica bien entre roles (diseño, backend, datos, infraestructura)

## 🇺🇸 El Script en Inglés (Para la entrevista)

> *"Being a fullstack developer means being able to contribute meaningfully across all layers of a web application—the frontend that users interact with, the backend APIs that process business logic, the database that stores persistent data, and the glue that connects them all. What it doesn't mean is being an expert in every technology simultaneously. The real value of fullstack is the ability to build a complete feature end-to-end without being blocked by layer boundaries—from designing the UI, to writing the API endpoint, to modeling the database schema, to deploying the change. In practice, most fullstack developers have a stronger area—mine is backend and data integration—but I can navigate and contribute across the full stack."*

## 🔑 Vocabulario Clave

* **Full stack:** Conjunto completo de capas de una aplicación web
* **End-to-end feature:** Funcionalidad completa de interfaz a base de datos
* **Layer boundary:** Límite entre capas de la aplicación
* **Business logic:** Lógica del negocio implementada en el backend
* **Deployment:** Proceso de desplegar la aplicación en producción
